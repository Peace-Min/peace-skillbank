# EMPTY_CATCH_BLOCK — 빈 catch 블록

- **건수**: 89  |  **심각도**: 높음  |  **트랙**: C
- **Sparrow 설명**: 빈 예외 처리 블록 체커는 예외를 처리하는 코드 내용이 없는 예외 처리 블록을 검출합니다.

## 지켜야 할 규칙 (무엇을 왜 검출)
예외를 조용히 삼키면 **결함이 은폐**되어 잘못된 상태로 실행이 계속된다. 방산 신뢰성시험에서 예외처리 누락은 핵심 지적 사항 — 오류가 로그에도 남지 않으면 사후 원인분석(사고조사)이 불가능하다. "정상적으로 무시해도 되는 예외"조차 **의도를 코드/주석으로 남겨야** 무분별한 삼킴이 아니라 설계로 인정된다.

## 표준 매핑 (교차참조)
- CWE: **CWE-390** (Detection of Error Condition Without Action); 연관 **CWE-391**(Unchecked Error Condition)
- 무기체계 보안약점 점검 목록: **187 항목 100% 적용(설정 확정)**; 개별 항목번호는 Sparrow 체커↔표준 매핑 추출 후 기입(대기)
- 행안부 SW보안약점(2021): "오류상황 대응 부재"

## 진성 판별 기준
- `catch { }` 또는 `catch (Exception) { }` 본문이 **비어 있음**(또는 주석만).
- 삼킨 예외에 대한 **로깅·상태복구·상위 전파·사용자 통지 중 아무것도 없음**.
- 특히 `catch (Exception)`/`catch` 광범위 포착과 결합되면 위험 가중(→ [[OVERLY_BROAD_CATCH]] 병행 검토).

## 이렇게 보여도 넘기지 말 것
아래처럼 보여도 **스킵하지 말고**, 조건을 확인해 처리/수정한다(전건 수정).
- **의도된 무시처럼 보이는 경우**: 예로 `int.TryParse` 대체 목적의 협소한 `catch (FormatException) { /* 기본값 사용 */ }` — 빈 본문이면 여전히 지적 대상이다. **좁은 예외형 + 사유 주석 + 최소 로깅**을 남기도록 수정한다.
- Dispose/Cleanup 경로에서 이미 실패한 자원 해제 예외 무시(관례)처럼 보여도 — 최소한 로깅을 남긴다.
- EMPTY_CATCH 는 "정말 무시가 옳다"도 **의도 주석 + 좁은 예외형 + 로깅**으로 바꾸는 것이 표준. 무시 자체를 근거로 건너뛰지 않는다.


## LLM 판단에 필요한 필수 문맥
- catch 블록 전체와 블록 안의 주석/전처리 조건 포함 여부.
- 해당 try 본문이 수행하는 작업과 예외가 발생했을 때 복구/무시가 업무적으로 허용되는지.
- 주변 finally/cleanup 코드와 상위 호출자가 예외를 별도로 처리하는지.
- catch가 넓은 예외(`catch`, `catch (Exception)`)인지 구체 예외인지.

## 문맥 부족 시 보류 기준
- `catch { }` 또는 주석만 있는 catch는 원칙적으로 진성 후보이나, 의도적 무시가 명확히 문서화되어 있는지 알 수 없으면 보류한다.
- cleanup 실패 무시처럼 허용 가능한 패턴인지 판단할 수 없으면 `needs_context=true`로 둔다.

## 추가로 요청해야 할 코드 범위
- 검출 catch를 포함하는 전체 try/catch/finally 구문.
- 해당 메서드 전체.
- 무시가 의도된 cleanup/API 호출이면 호출 API와 오류 정책 주석.

## 수정 패턴 (C# 예시)
```csharp
// Before — 삼킴
try { DoWork(); }
catch { }                       // 오류 은폐

// After (A) 로깅 후 정책에 맞게 (재던지기/기본값)
try { DoWork(); }
catch (Exception ex)
{
    _log.Error("DoWork 실패", ex);   // 프로젝트 로깅 사용
    throw;                            // 복구 불가면 전파
}

// After (B) 예외형을 좁히고 의도 명시 (무시가 정말 정상일 때)
try { file.Delete(); }
catch (IOException ex)
{
    // 이미 삭제됨/잠금 — 정리 실패는 무시해도 안전
    _log.Debug("임시파일 삭제 실패(무시)", ex);
}
```
- 기본 방침: **좁은 예외형 + 로깅**. 복구 가능하면 복구, 아니면 `throw;`(스택 보존, `throw ex;` 금지). 무시가 정당하면 **좁힌 예외형 + 사유 주석 + 최소 로깅**.
