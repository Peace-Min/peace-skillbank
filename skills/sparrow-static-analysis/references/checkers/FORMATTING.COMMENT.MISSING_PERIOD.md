# FORMATTING.COMMENT.MISSING_PERIOD — 주석 종결 문장부호 누락

- **건수**: (실측 시 변동)  |  **심각도**: 낮음  |  **트랙**: B
- **Sparrow 설명**: 주석 문장의 끝에 종결 문장 부호를 써주어야 합니다.

> **대부분 Track A(SparrowSyntaxFix/SparrowSyntaxFix) 또는 Track B(SparrowCommentFix/SparrowCommentFix layout)가 자동 처리한다. 이 가이드는 도구가 못 고친 잔여를 LLM이 처리할 때 사용**한다.

## 지켜야 할 규칙 (무엇을 왜 검출)
주석 문장은 마침표(`.`)·물음표(`?`)·느낌표(`!`) 등 **종결 문장부호로 끝나야** 한다. **보안 결함이 아니라 스타일·가독성·표준(MSDN C# 코딩 규칙) 준수** 항목이다. 종결 부호가 있으면 주석이 완결된 문장임이 드러나 판독성이 좋아진다. Sparrow는 종결 부호 없이 끝나는 주석문을 검출한다.

## 표준 매핑 (교차참조)
- MSDN C#:2015 코딩 규칙 **3.3** — 주석 텍스트 끝에는 마침표를 붙입니다.
- C# Coding conventions : 2023 — 주석 텍스트 끝에는 마침표를 붙입니다.
- CWE: 스타일 항목이라 직접 매핑 없음(광의 **CWE-1078** 계열).
- 무기체계 보안약점 점검 목록: **187 항목 100% 적용**; 개별 항목번호는 Sparrow 체커↔표준 매핑 추출 후 기입(대기).

## 진성 판별 기준
- 주석문(`//` 한 줄 또는 `/* */` 블록)의 문장이 종결 부호(`.`, `?`, `!`) 없이 끝난다.
- 예: `// BadCase1` — 진성. `// GoodCase1.` — 정상.

## 이렇게 보여도 넘기지 말 것
스타일 항목이라도 **전건 수정**한다. 짧은 라벨/키워드성 주석이라도 스킵하지 말고 종결 부호를 붙인다.
- 코드 조각을 잠시 비활성화한 **주석 처리된 코드**(`// return x;`)나 URL·경로처럼 마침표가 어색한 내용이면, 문장이 아니라 코드/토큰이므로 억지로 마침표를 붙이지 말고 그 근거(문장 아님)를 판정에 기록한다. 서술형 주석이면 종결 부호를 붙인다.
- 여러 줄로 이어지는 한 문장이면 **마지막 줄에만** 종결 부호를 둔다.
- 종결 부호 생략이 관례처럼 보여도 표준은 종결 부호 포함이다 — 근거 확인 후 수정한다.

## 수정 패턴 (C# 예시)
```csharp
// Before
// BadCase1
Console.WriteLine("BadCase1");

// After
// GoodCase1.
Console.WriteLine("GoodCase1");
```

## 검증 확인 조건 (G2)
- 빌드 통과(net472 / C# 7.3) — 주석 변경은 의미에 영향 없음.
- 재분석 시 해당 라인 FORMATTING.COMMENT.MISSING_PERIOD 소멸.
- 신규 검출 0 — 주석만 수정하고 코드 토큰을 건드리지 않았는지 확인.
