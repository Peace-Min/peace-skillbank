# MISSING_BLANK_LINE_BEFORE_COMMENT — 누락된 주석 앞 빈 줄

- **건수**: (실측 시 변동)  |  **심각도**: 낮음  |  **트랙**: B
- **Sparrow 설명**: 주석은 코드의 끝이 아니라 별도의 줄을 할당하여 작성해야 합니다.

> **대부분 Track A(SparrowSyntaxFix/SparrowSyntaxFix) 또는 Track B(SparrowCommentFix/SparrowCommentFix layout)가 자동 처리한다. 이 가이드는 도구가 못 고친 잔여를 LLM이 처리할 때 사용**한다.

## 지켜야 할 규칙 (무엇을 왜 검출)
주석은 코드 줄 끝에 꼬리로 붙이지 말고 **별도의 줄**에 작성하며, 앞 코드와 시각적으로 분리되도록 **주석 앞에 빈 줄**을 둔다. **보안 결함이 아니라 스타일·가독성·표준 준수** 항목이다. 코드 바로 아래 줄에 주석이 붙어 있으면 어느 코드에 대한 설명인지, 새 블록의 서두인지 경계가 흐려진다. Sparrow는 앞 코드 줄 바로 다음에 빈 줄 없이 시작하는 (설명 블록성) 주석 줄을 검출한다.

## 진성 판별 기준
- 코드 문장 바로 다음 줄에 빈 줄 없이 별도 줄 주석(`//` 또는 `/* */`)이 시작된다(뒤 코드 블록을 설명하는 주석인데 앞 코드와 붙어 있음).
- 꼬리 주석(코드와 같은 줄 끝의 `// ...`)을 별도 줄로 올리라는 취지도 포함될 수 있다(규칙 정의 기준).

## 이렇게 보여도 넘기지 말 것
스타일 항목이라도 **전건 수정**한다. 주석이 앞 코드에 붙어 있는 게 자연스러워 보여도 앞에 빈 줄을 넣는다.
- 블록 **첫 줄**의 주석(여는 `{` 바로 다음 줄 등)은 앞에 빈 줄이 불필요할 수 있으므로, 규칙이 요구하는 위치(블록 내부 첫 줄 예외 여부)를 확인한 뒤 처리한다(문맥 확보).
- 연속된 여러 줄 주석 묶음은 묶음 **앞**에 한 번만 빈 줄을 두고, 묶음 내부에는 빈 줄을 넣지 않는다.
- 붙은 주석이 관례처럼 보여도 표준은 주석 앞 빈 줄 분리다 — 근거 확인 후 수정한다.

## 수정 패턴 (C# 예시)
```csharp
// Before — 앞 코드 바로 다음 줄에 붙은 설명 주석
DoFirstThing();
// Prepare the next step.
DoSecondThing();

// After — 주석 앞에 빈 줄로 분리
DoFirstThing();

// Prepare the next step.
DoSecondThing();
```
