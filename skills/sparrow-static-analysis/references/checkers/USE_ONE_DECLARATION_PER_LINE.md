# USE_ONE_DECLARATION_PER_LINE — 한 줄에 작성된 여러 개의 선언(방위사업청)

- **건수**: (실측 시 변동)  |  **심각도**: 낮음  |  **트랙**: B
- **Sparrow 설명**: 한 줄에 하나의 선언문만 있어야 합니다.

> **대부분 Track A(dotnet format/SparrowSyntaxFix) 또는 Track B(SparrowCommentFix/dotnet format whitespace)가 자동 처리한다. 이 가이드는 도구가 못 고친 잔여를 LLM이 처리할 때 사용**한다.

## 지켜야 할 규칙 (무엇을 왜 검출)
한 줄에는 **선언을 하나만** 작성해야 한다. **보안 결함이 아니라 스타일·가독성·표준 준수** 항목이다. `int a, b, c;` 또는 `int x = 0; int y = 1;`처럼 한 줄에 여러 변수를 몰아 선언하면 각 변수의 초기값·타입·용도 파악이 어렵고 줄 단위 diff/디버깅이 나빠진다. Sparrow는 한 줄에 둘 이상의 선언이 있는 경우를 검출한다.

## 표준 매핑 (교차참조)
- **Rule 원문 미확보 — 확보 시 레퍼런스 보강.** 아래는 도메인 지식 기반 참고 매핑이다.
- MSDN C# 코딩 규칙(한 줄 한 문장·선언 분리) / 방위사업청 시큐어코딩(무기체계) 가독성 지침 — 한 줄 한 선언.
- CWE: 스타일 항목이라 직접 매핑 없음(광의 **CWE-1078** 계열).
- 무기체계 보안약점 점검 목록: **187 항목 100% 적용**; 개별 항목번호는 Sparrow 체커↔표준 매핑 추출 후 기입(대기).

## 진성 판별 기준
- 한 줄에 콤마로 묶인 다중 변수 선언(`int a, b, c;`)이 있거나, 세미콜론으로 구분된 여러 선언(`int x = 0; string s = "";`)이 한 줄에 있다.

## 이렇게 보여도 넘기지 말 것
스타일 항목이라도 **전건 수정**한다. 관련 변수를 한 줄에 묶은 게 간결해 보여도 선언을 줄마다 나눈다.
- `for (int i = 0, n = list.Count; i < n; i++)`의 **for 초기화절 다중 선언**은 for 헤더 구성 요소이므로 나누지 않는다(대상 아님) — 근거를 판정에 기록한다.
- 콤마 다중 선언을 나눌 때 **각 변수의 타입을 정확히 복제**한다. 특히 배열/포인터성 표기에 주의하고, 초기화가 없던 변수는 초기화 없이 그대로 분리한다.
- 튜플 분해 선언(`var (a, b) = ...;`)은 단일 선언문이므로 대상이 아니다 — 대상 아님을 판정에 기록한다.
- 한 줄 다중 선언이 관례처럼 보여도 표준은 한 줄 한 선언이다 — 근거 확인 후 수정한다.

## 수정 패턴 (C# 예시)
```csharp
// Before — 한 줄에 여러 선언
int a, b, c;
int x = 0; string s = "";

// After — 한 줄 한 선언
int a;
int b;
int c;
int x = 0;
string s = "";
```

## 검증 확인 조건 (G2)
- 빌드 통과(net472 / C# 7.3).
- 재분석 시 해당 라인 USE_ONE_DECLARATION_PER_LINE 소멸.
- 신규 검출 0 — 선언 분리로 각 변수의 타입/초기값이 정확히 보존됐는지 확인.
