# USE_ONE_STATEMENT_PER_LINE — 한 줄에 작성된 여러 개의 구문(방위사업청)

- **건수**: (실측 시 변동)  |  **심각도**: 낮음  |  **트랙**: B
- **Sparrow 설명**: 한 줄에 하나의 문장만 사용해야 합니다.

> **대부분 Track A(SparrowSyntaxFix/SparrowSyntaxFix) 또는 Track B(SparrowCommentFix/SparrowCommentFix layout)가 자동 처리한다. 이 가이드는 도구가 못 고친 잔여를 LLM이 처리할 때 사용**한다.

## 지켜야 할 규칙 (무엇을 왜 검출)
한 줄에는 **문장을 하나만** 작성해야 한다. **보안 결함이 아니라 스타일·가독성·표준(MSDN C# 코딩 규칙) 준수** 항목이다. `A(); B();`처럼 세미콜론으로 여러 문장을 한 줄에 몰아 쓰면 판독성·디버깅(줄 단위 브레이크포인트)·diff 가독성이 나빠진다. Sparrow는 한 줄에 둘 이상의 문장이 있는 경우를 검출한다.

## 표준 매핑 (교차참조)
- MSDN C#:2015 코딩 규칙 **2.2** — 문을 한 줄에 하나씩만 작성합니다.
- C# Coding conventions : 2023 — 문을 한 줄에 하나씩만 작성합니다.
- 방위사업청 시큐어코딩(무기체계) 가독성 지침 — 한 줄 한 문장.
- CWE: 스타일 항목이라 직접 매핑 없음(광의 **CWE-1078** 계열).
- 무기체계 보안약점 점검 목록: **187 항목 100% 적용**; 개별 항목번호는 Sparrow 체커↔표준 매핑 추출 후 기입(대기).

## 진성 판별 기준
- 한 물리적 줄에 세미콜론으로 끝나는 실행 문장이 둘 이상 있다.
- 예: `Console.WriteLine("A"); Console.WriteLine("B");` — 진성.

## 이렇게 보여도 넘기지 말 것
스타일 항목이라도 **전건 수정**한다. 짧은 문장을 한 줄에 몰아 쓴 것이 간결해 보여도 줄을 나눈다.
- `for (init; cond; iter)`의 세미콜론은 **for 헤더의 구성 요소**이지 별개 문장이 아니므로 나누지 않는다(대상 아님). 이 경우 근거(for 헤더)를 판정에 기록한다.
- 한 줄에 여러 선언이 있는 경우는 이 체커가 아니라 `USE_ONE_DECLARATION_PER_LINE` 대상이니 혼동하지 말고 해당 규칙으로 처리한다.
- 문장을 나눌 때 각 문장의 들여쓰기를 블록 수준에 맞추고, 부수효과 순서를 그대로 보존한다.
- 한 줄 몰아쓰기가 관례처럼 보여도 표준은 한 줄 한 문장이다 — 근거 확인 후 수정한다.

## 수정 패턴 (C# 예시)
```csharp
// Before — 한 줄에 두 문장
Console.WriteLine("BadCase1 - 1"); Console.WriteLine("BadCase1 - 2");

// After — 한 줄 한 문장
Console.WriteLine("GoodCase1 - 1");
Console.WriteLine("GoodCase1 - 2");
```

## 검증 확인 조건 (G2)
- 빌드 통과(net472 / C# 7.3).
- 재분석 시 해당 라인 USE_ONE_STATEMENT_PER_LINE 소멸.
- 신규 검출 0 — 문장 분리로 실행 순서·부수효과가 바뀌지 않았는지 확인.
