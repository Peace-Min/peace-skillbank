# Track C 수정 프롬프트 (fix-prompt)

## 지시 (모델/사람에게)

당신은 Sparrow(파수) 정적분석 **Track C** 검출 1건을 **수정한다**. 이 검출 라인은 **반드시 고쳐야 할 결함**이다.
당신의 임무는 진위를 "판정"하는 것이 아니라, 검출 라인의 결함을 제거하는 **수정 코드(before/after)를 만드는 것**이다.

> **전건(全件) 수정 정책** — Sparrow가 잡은 라인은 예외 없이 전부 고친다.
> "true positive / 위양성 / 정상이라 필요없음" 같은 판정으로 수정을 건너뛰지 마라. **검출되었다는 것 자체가 수정 대상**이라는 뜻이다.
> 지금 스니펫만으로 안전하게 못 고치는 경우에만 `보류`로 두되, 보류는 "안 함"이 아니라 **문맥 확보 후 반드시 수정**하는 대기 상태다.

규칙:

1. **기본은 `수정`이다.** 아래 **[검출 항목]** 의 검출 라인을, **[체커 가이드]** 의 `## 수정 패턴 (C# 예시)`를 따라
   **.NET Framework 4.7.2 / C# 7.3** 문법으로 고쳐 `fix.before`/`fix.after`를 채워라.
   - `using (...) { }` **블록** 문법만(using 선언 금지), switch 식·nullable 참조형 금지.
   - `after`는 검출 소멸이 목표이되, 조용한 무시(빈 catch, 무의미한 널병합)로 결함을 은폐하지 마라.
2. **수정 대상 라인을 정확히 집어라.** 항목에 **`## 수정 대상`** 이 있으면 그 파일/라인/대상 코드를 최우선으로 본다.
   `TARGET LINE` 표시가 있으면 `fix.lines`는 그 라인을 포함해야 하며, 표시 없는 주변 코드는 문맥용이라 별도 근거 없이 임의 수정하지 마라.
3. **근거는 가이드에서 인용하라.** 어떻게 고칠지의 근거는 [체커 가이드]의 `## 결함 판별 기준` 또는
   `## 이렇게 보여도 넘기지 말 것`에서 인용한다. 가이드에 없는 일반론으로 처리하지 마라.
4. **지어내지 마라 → 보류.** 소스가 잘려 null/자원 근원이 이 스니펫 밖(외부 경로)이라 안전한 수정을 만들 수 없으면
   `verdict = 보류`, `needs_context = true`, `missing_context`에 필요한 파일/심볼/코드 범위를 적어라.
   자료는 충분한데 로컬 모델이 판단하기 벅찬 경우에만 `needs_frontier = true`로 둔다.
   **보류는 스킵이 아니다** — 문맥 확보 후 반드시 수정한다.
5. `cwe`는 가이드 `## 표준 매핑`의 CWE를 그대로 기입(예: `CWE-476`). `weapon_item`은 187 매핑 추출 전이므로
   `미매핑(187 추출 후 기입)`으로 둔다.
6. **출력은 아래 JSON 스키마 객체 하나만.** 다른 텍스트·마크다운·설명을 앞뒤에 붙이지 마라.

### 출력 JSON 스키마 (키를 정확히 사용)

```json
{
  "id": "string", "checker": "string", "file": "string", "line": "string",
  "verdict": "수정|보류",
  "rationale": "가이드 기준을 인용한 수정 근거",
  "fix": { "lines": "string", "before": "C# 코드", "after": "C# 코드" },
  "hold_reason": "보류일 때만",
  "needs_context": false,
  "missing_context": ["필요한 추가 문맥"],
  "needs_frontier": false,
  "cwe": "CWE-476", "weapon_item": "미매핑(187 추출 후 기입)"
}
```

채움 규칙:
- **수정**(기본) → `fix.lines`/`fix.before`/`fix.after`를 채우고, `hold_reason`은 `""`.
- **보류**(문맥 부족일 때만) → `hold_reason` 채우고, `needs_context=true` + `missing_context`를 채운다.
  자료는 충분하지만 상위 모델 판단이 필요하면 `needs_frontier=true`. `fix`는 비운다.
  (보류는 스킵이 아니라 문맥 확보 후 반드시 수정하는 대기 상태다.)

`id`/`checker`/`file`/`line`은 [검출 항목]의 표(`| 필드 | 값 |`)와 H1(`# 체커키 @ 파일:라인`)에서 그대로 옮긴다.
