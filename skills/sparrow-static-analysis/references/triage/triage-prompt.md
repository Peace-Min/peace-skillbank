# Track C 수정 작업 프롬프트

> 이 파일은 `Run-Triage.ps1 prepare`와 통합 GUI의 Track C 탭이 사용하는 요청 템플릿이다.
> `{{GUIDE}}`는 해당 체커 가이드 전문으로, `{{ITEM}}`은 Sparrow 검출 항목 md 전문으로 치환된다.
> 생성된 `requests/<체커>/<ID>_<체커>.md` 하나가 곧 LLM/사람에게 전달할 self-contained 작업 요청이다.

---

## 역할

당신은 Sparrow(파수) 정적분석 Track C 검출 1건을 실제 코드 수정으로 연결하는 작업자다.
아래 **[체커 가이드]** 와 **[검출 항목]** 을 읽고, 대상 코드에서 무엇을 고쳐야 하는지 바로 실행 가능한 수정 지시서를 작성한다.

이 프로젝트는 전건 수정 정책이다. false-positive로 버리지 않는다. 지금 문맥만으로 수정할 수 없으면 스킵하지 말고 필요한 추가 문맥을 명시한다.

## 작업 규칙

1. 반드시 [체커 가이드]의 `## 진성 판별 기준`, `## 이렇게 보여도 넘기지 말 것`, `## 수정 패턴 (C# 예시)`를 기준으로 판단한다.
2. [검출 항목]의 `## 소스 코드`와 `## 수정 대상`을 우선 확인한다. `TARGET LINE`이 있으면 그 라인을 포함한 최소 범위만 고친다.
3. C# 코드는 .NET Framework 4.7.2 / C# 7.3 문법으로 작성한다.
   - `using (...) { }` 블록 문법만 사용한다.
   - using 선언, switch 식, nullable reference type 문법은 사용하지 않는다.
4. 조용한 무시, 빈 catch, 의미 없는 null 병합처럼 결함을 덮는 수정은 금지한다.
5. 소스가 잘렸거나 호출부/소유권/예외 후보 등 문맥이 부족하면 임의 수정하지 말고 `추가 필요 문맥`에 필요한 파일, 심볼, 코드 범위를 적는다.
6. 출력은 Markdown 수정 지시서로 작성한다. JSON 판정 파일은 작성하지 않는다.

## 출력 형식

아래 형식을 유지한다.

```markdown
# <체커키> 수정 지시

## 대상
- ID:
- 파일:
- 라인:
- 체커:

## 수정 여부
- 상태: 수정 가능 | 문맥 필요
- 근거: <체커 가이드 기준을 짧게 인용/요약>

## 수정 지시
<실제 파일에서 어느 범위를 어떻게 바꿀지 구체적으로 작성>

## Before
```csharp
<수정 전 최소 코드>
```

## After
```csharp
<수정 후 최소 코드>
```

## 추가 필요 문맥
- <없으면 "없음">

## 검증
- 빌드:
- Sparrow 재분석:
- 확인할 체커:
```
